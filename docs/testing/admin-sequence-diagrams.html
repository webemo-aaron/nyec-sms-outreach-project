Admin Driven Workflow

Easy-to-Read Sequence Diagrams

These diagrams describe the admin/operator experience for local validation and live-readiness. They intentionally avoid customer-facing pages and patient-facing UI.

Primary User

Admin/operator managing cohorts, campaigns, dispatches, opt-outs, and evidence.

Main System

Vue admin UI calling the local Node operational API.

External System

Twilio for test sends, dispatch sends, status callbacks, and inbound STOP/START.

Evidence

JSON store, audit events, dispatch rows, message rows, opt-outs, and billing summary.

1. Admin Startup and Health Check

The admin starts local services, confirms health, and opens the operational Vue UI.

sequenceDiagram
  autonumber
  actor Admin
  participant Shell as Local Shell
  participant API as Node API
  participant UI as Vue Admin UI
  participant Store as JSON Store

  Admin->>Shell: Start API and UI
  Shell->>API: npm run start
  API->>Store: Load persisted state
  Shell->>UI: npm run dev or Docker UI
  Admin->>API: GET /health
  API-->>Admin: ok
  Admin->>UI: Open /command-center
  UI->>API: GET /api/nyec/dashboard
  API-->>UI: Current operational summary
  UI-->>Admin: Command center shows live API status

2. Cohort Import and Review

The admin imports a cohort, reviews accepted/rejected rows, and decides whether the cohort is usable.

sequenceDiagram
  autonumber
  actor Admin
  participant UI as Vue Admin UI
  participant API as Node API
  participant Store as JSON Store

  Admin->>UI: Upload or paste cohort CSV
  UI->>API: POST /api/nyec/mef/batches
  API->>API: Parse CSV and validate phone/facility fields
  API->>Store: Persist batch, accepted rows, rejected preview
  API->>Store: Write MEF import audit event
  API-->>UI: Batch id, accepted count, rejected count
  UI-->>Admin: Show cohort summary and rejected row preview
  Admin->>UI: Select usable cohort for campaign setup
Admin UI labels this as cohort management. Internal MEF naming may remain in API paths, but it should not drive the admin experience.

3. Campaign Create, Review, and Launch

The admin creates a campaign from an imported cohort, reviews the operational settings, and launches it.

sequenceDiagram
  autonumber
  actor Admin
  participant UI as Vue Admin UI
  participant API as Node API
  participant Store as JSON Store

  Admin->>UI: Enter campaign name, facility, NPI, limit, SMS body
  UI->>API: POST /api/nyec/campaigns
  API->>Store: Persist draft campaign
  API->>Store: Write CAMPAIGN_CREATED audit event
  API-->>UI: Campaign id and draft details
  UI-->>Admin: Show campaign detail
  Admin->>UI: Launch campaign
  UI->>API: POST /api/nyec/campaigns/{id}/launch
  API->>Store: Mark campaign RUNNING
  API->>Store: Write CAMPAIGN_LAUNCHED audit event
  API-->>UI: Campaign status RUNNING
  UI-->>Admin: Campaign is ready for dispatch

4. Manual Dispatch and Message Rows

The admin triggers dispatch. The API applies daily limits, excludes opt-outs, and records outbound messages.

sequenceDiagram
  autonumber
  actor Admin
  participant UI as Vue Admin UI
  participant API as Node API
  participant Store as JSON Store
  participant Twilio

  Admin->>UI: Run dispatch
  UI->>API: POST /api/nyec/campaigns/{id}/dispatches
  API->>Store: Load campaign, cohort recipients, prior sends, opt-outs
  API->>API: Apply daily limit and eligibility rules
  alt Twilio live configured
    API->>Twilio: Send SMS for each eligible recipient
    Twilio-->>API: Message SID and queued status
  else Test/simulated mode
    API->>API: Generate simulated message SID
  end
  API->>Store: Persist dispatch batch and outbound messages
  API->>Store: Write dispatch and billing evidence
  API-->>UI: Dispatch summary and message rows
  UI-->>Admin: Show queued/sent/failed counts

5. Twilio Status Callback

Twilio reports delivery state. The admin can inspect message status and audit evidence.

sequenceDiagram
  autonumber
  participant Twilio
  participant API as Node API
  participant Store as JSON Store
  actor Admin
  participant UI as Vue Admin UI

  Twilio->>API: POST /api/nyec/sms/status
  API->>Store: Find outbound message by SID
  API->>Store: Update provider/callback status
  API->>Store: Write TWILIO_STATUS_CALLBACK audit event
  Admin->>UI: Open Send Activity
  UI->>API: GET /api/nyec/outbound-messages
  API-->>UI: Message rows with callback status
  UI-->>Admin: Show delivered/failed/undelivered state

6. STOP and START Opt-Out Handling

Inbound STOP suppresses future sends. START clears the active opt-out when allowed by policy.

sequenceDiagram
  autonumber
  participant PatientPhone as Recipient Phone
  participant Twilio
  participant API as Node API
  participant Store as JSON Store
  actor Admin
  participant UI as Vue Admin UI

  PatientPhone->>Twilio: Reply STOP
  Twilio->>API: POST /api/nyec/sms/inbound
  API->>Store: Create active opt-out for phone
  API->>Store: Write TWILIO_INBOUND_CALLBACK audit event
  Admin->>UI: Open opt-out review
  UI->>API: GET /api/nyec/opt-outs
  API-->>UI: Active opt-out list
  UI-->>Admin: Phone appears as opted out

  PatientPhone->>Twilio: Reply START
  Twilio->>API: POST /api/nyec/sms/inbound
  API->>Store: Remove active opt-out
  API->>Store: Write re-subscribe audit event
  UI->>API: GET /api/nyec/opt-outs
  API-->>UI: Active opt-out list without phone
  UI-->>Admin: Phone no longer appears as opted out

7. No-Credit Twilio Test Credentials

The admin validates Twilio API connectivity with test credentials and magic From number before any real SMS test.

sequenceDiagram
  autonumber
  actor Admin
  participant UI as Vue Admin UI
  participant API as Node API
  participant Twilio as Twilio Test API
  participant Store as JSON Store

  Admin->>UI: Configure test SID/token and From +15005550006
  UI->>API: PUT /api/nyec/twilio/config
  API->>Store: Persist masked Twilio config
  Admin->>UI: Send test SMS
  UI->>API: POST /api/nyec/twilio/test
  API->>Twilio: POST /Messages.json with test credentials
  Note over API,Twilio: No real SMS is sent and no message credits are used
  Twilio-->>API: queued SM... response
  API->>Store: Write TWILIO_TEST_SMS_SENT audit event
  API-->>UI: Provider Twilio, status queued
  UI-->>Admin: Test path accepted
Use https://example.com or another public placeholder for test credential StatusCallback. Twilio rejects localhost callback URLs.

8. Audit, Billing, and Reset

The admin reviews operational evidence, then resets local test state when validation is complete.

sequenceDiagram
  autonumber
  actor Admin
  participant UI as Vue Admin UI
  participant API as Node API
  participant Store as JSON Store

  Admin->>UI: Open Audit
  UI->>API: GET /api/nyec/audit/events
  API-->>UI: Configuration, import, campaign, dispatch, callback, opt-out events
  Admin->>UI: Open Billing
  UI->>API: GET /api/nyec/billing/summary
  API-->>UI: Billable messages and cost estimate
  Admin->>UI: Reset local test data
  UI->>API: POST /api/nyec/admin/reset
  API->>Store: Clear local cohorts, campaigns, dispatches, messages, opt-outs
  API-->>UI: Reset summary
  UI-->>Admin: Local state is clean for next validation run
Sequence diagrams are admin/operator focused. They do not define customer-facing UI pages.
