Local Release Gate

NYeC SMS Outreach Operational Workflow

This page documents the designed local validation workflow before laptop handoff: Node API, Vue UI, synthetic operational data, Twilio test credentials, Docker packaging, and the work-laptop pull/run path.

Data Policy Synthetic data only

No real member data is needed for local or Docker validation.

Twilio Policy No-credit test path

Use Twilio test credentials with magic From number +15005550006.

Handoff Policy Verify, then pull main

All repo gates land on main before the work laptop pulls.

End-to-End Local Gate

The local script verifies code health, starts the API/UI, drives an operational workflow, checks UI routes, optionally probes Twilio, and resets state. 

flowchart TD
  A[Start on main] --> B[Install and test local-api]
  B --> C[Run Vue UI tests]
  C --> D[Build Vue production assets]
  D --> E[Start local API on 127.0.0.1:3001]
  E --> F[Start Vue UI on 127.0.0.1:5173]
  F --> G[Reset local API state]
  G --> H[Import synthetic MEF CSV]
  H --> I[Create campaign from accepted rows]
  I --> J[Launch campaign]
  J --> K[Run manual dispatch]
  K --> L[Simulate Twilio delivered callback]
  L --> M[Simulate STOP opt-out]
  M --> N[Simulate START resubscribe]
  N --> O[Verify dashboard, billing, audit, messages]
  O --> P[Check operational UI routes]
  P --> Q[Optional Twilio test-send]
  Q --> R[Final reset]
  R --> S[Local gate complete]

  B -. fail .-> X[Stop and fix API]
  C -. fail .-> Y[Stop and fix UI contracts/routes]
  K -. fail .-> Z[Stop and inspect campaign/store logic]
          
./scripts/local-operational-validation.sh

No-Credit Twilio Test Path

This validates Twilio request construction and credentials without sending a real carrier SMS. Local callback behavior is validated separately by posting directly to the local API. 

sequenceDiagram
  autonumber
  participant Env as local-api/.env
  participant API as Local Node API
  participant Twilio as Twilio Test Credentials
  participant Store as Local JSON Store

  Env->>API: Test SID, test auth token, From +15005550006
  Env->>API: Callback base https://example.com
  API->>Twilio: POST /Messages.json
  Note over API,Twilio: Test credentials + magic From number use no SMS credits
  Twilio-->>API: queued response with SM... SID
  API->>Store: Audit TWILIO_TEST_SMS_SENT
  API-->>Env: ok=true, provider=Twilio, status=queued

  Env->>API: POST /api/nyec/sms/status with generated or simulated SID
  API->>Store: Update outbound callback status
  Env->>API: POST /api/nyec/sms/inbound STOP / START
  API->>Store: Create or clear active opt-out
          
TWILIO_FROM_NUMBER=+15005550006
TWILIO_MESSAGING_SERVICE_SID=
TWILIO_CALLBACK_BASE_URL=https://example.com

curl -i -H 'content-type: application/json' \
  -d '{"to":"+14108675310","body":"NYeC local validation test"}' \
  http://127.0.0.1:3001/api/nyec/twilio/test
Twilio rejects http://localhost:3001 as a public StatusCallback URL. Use a harmless public placeholder for test credentials, then simulate callbacks locally.

Docker Packaging Path

Docker validation uses the same API/UI workflow after building packaged images. On managed laptops, the CA script injects the work root certificate and pins npm for proxy stability. 

flowchart LR
  A[Set DOCKER_CLI for Windows Docker Desktop] --> B[Run docker-env-check]
  B --> C{Docker reachable?}
  C -- no --> D[Fix Docker Desktop context or WSL integration]
  C -- yes --> E[Build API image]
  E --> F[Build UI image]
  F --> G[Tag Compose images and convenience latest tags]
  G --> H[docker compose up local-api vue-ui]
  H --> I[Wait for /health endpoints]
  I --> J[Run synthetic operational API workflow]
  J --> K[Check Vue routes]
  K --> L[Final reset]
  L --> M[Docker gate complete]

  E -. TLS failure .-> N[Use docker-build-with-ca.sh with work root CA]
  F -. npm proxy failure .-> O[npm 9.8.1 pin and NODE_ENV=development in UI build stage]
          
export DOCKER_CLI="/mnt/c/Program Files/Docker/Docker/resources/bin/docker"
./scripts/docker-env-check.sh
./scripts/docker-build-with-ca.sh ~/ZscalerRootCertificate-2048-SHA256-Feb2025.crt
docker compose up -d local-api vue-ui

Merge and Work-Laptop Pull

Completed changes are verified on the development branch, fast-forward merged into main, verified again, pushed, then pulled on the work laptop. 

flowchart TD
  A[Feature branch work] --> B[Run branch gates]
  B --> C{All gates pass?}
  C -- no --> D[Fix branch]
  D --> B
  C -- yes --> E[Switch to main]
  E --> F[Pull origin/main]
  F --> G[Fast-forward merge feature branch]
  G --> H[Run main gates]
  H --> I{Main gates pass?}
  I -- no --> D
  I -- yes --> J[Push origin main]
  J --> K[Work laptop git switch main]
  K --> L[git pull --ff-only origin main]
  L --> M[Rebuild/run local or Docker workflow]
  M --> N[Open /command-center and confirm Operational Vue UI]
          
git switch main
git pull --ff-only origin main
./scripts/local-operational-validation.sh
